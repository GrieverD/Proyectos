```
// 01 · Arquitectura

Estructura del Proyecto

El sitio vive en tu servidor en la siguiente ruta. Así está organizado:

Árbol de directorios bash 
/mnt/Maindeck/stacks/mitechoy/astro-site/
```

├── src/
│   └── pages/

```
// 02 · Despliegue

Desplegar Cambios

Cuando edites index.astro, sube los cambios con estos comandos desde tu servidor:

Flujo de despliegue completo bash 
# 1. Ve a tu directorio del proyecto
```

cd /mnt/Maindeck/stacks/mitechoy/astro-site

# 2. Haz respaldo rápido
cp src/pages/index.astro src/pages/index.astro.bak

# 3. Copia el nuevo index.astro (si lo subiste por SFTP)
cp ~/index.astro src/pages/index.astro

# 4. Compila el sitio
npm run build

# 5. (Opcional) Previsualiza antes de publicar
npm run preview

# 6. Sincroniza con Cloudflare Pages / servidor web
npx wrangler pages deploy dist/ –project-name=mitechoy
```
⚠️ Si usas caddy / nginx apuntando directamente a dist/, basta con ejecutar npm run build y el servidor sirve el nuevo contenido automáticamente.

// 03 · Dispositivos

Panel de Estado de Servicio

El botón flotante "Estado de servicio" abre un drawer lateral que muestra tus equipos en reparación en tiempo real. Los datos vienen de PocketBase; si PocketBase no está disponible usa datos de respaldo estáticos.

🔧
En proceso
LED verde · El equipo está siendo reparado activamente.
⏸️
En pausa
LED amarillo · Trabajo detenido temporalmente.
🚫
Detenido
LED rojo · Trabajo cancelado o sin continuación.
✅
Completado
LED azul · Reparación terminada y entregada.

// 04 · Dispositivos

Agregar Nuevos Dispositivos

Tienes dos métodos: via PocketBase (recomendado, sin tocar código) o editando el array estático en index.astro.

Método A — PocketBase (Admin Panel)
1

Abre el panel de PocketBase

Ve a https://pb.mitechoy.com/_/ e inicia sesión como admin.

2

Crea la colección service_orders (si no existe)

En el panel lateral haz clic en New collection. Nómbrala service_orders y agrega estos campos:

3

Campos requeridos de la colección

Ver tabla de campos a continuación.

CampoTipoDescripciónEjemplo
ticket_idtextID visible para el clienteMT-2026-0022
devicetextNombre del dispositivoXbox Series X
statusselectin_progress / paused / stopped / completedin_progress
imageurlURL de foto del equipo (opcional)https://…
createdautoFecha de creación (automático)—
4

Agrega un registro nuevo

Haz clic en + New record, llena los campos y guarda. El panel de estado del sitio lo mostrará al abrirse (hace fetch a PocketBase).

5

Configura las reglas de la colección como pública de lectura

En la colección ve a API Rules → List / View → deja vacío (sin regla = público). El resto déjalo bloqueado.

Método B — Editar el array estático en index.astro

Usa esto cuando PocketBase no está disponible o como respaldo.

SERVICE_DATA_FALLBACK en index.astro javascript 
// Busca esta línea en index.astro (línea ~1195):
```

const SERVICE_DATA_FALLBACK = [
// Copia y pega un bloque nuevo para cada equipo:
{
id: “MT-2026-0022”,        // ← Número de ticket
device: “MacBook Pro 2021”,  // ← Nombre del equipo
image: ””,                  // ← URL de foto (o vacío)
status: “in_progress”,      // ← in_progress|paused|stopped|completed
createdAt: “2026-04-01”,    // ← Fecha de recepción
completed: false             // ← true solo si status=“completed”
},
// … resto de equipos
];
```
🔑 Consejo: Para que PocketBase sea el único origen de verdad, actualiza solo desde el panel y deja el array estático como respaldo de emergencia.

// 05 · Dispositivos

Modificar el Estado de un Dispositivo

Desde PocketBase (recomendado)
1

Abre el registro

Ve a service_orders → busca el ticket del equipo → clic en el registro.

2

Cambia el campo status

Selecciona el nuevo estado en el dropdown: in_progress paused stopped completed

3

Guarda

El próximo usuario que abra el panel de estado verá el cambio al instante.

Flujo típico de estado
Ingresa
→
in_progress
→
paused
→
in_progress
→
completed: true
Por API (desde curl / scripts)
Actualizar estado vía API REST bash 
# Primero obtén el token de admin
```

curl -X POST https://pb.mitechoy.com/api/admins/auth-with-password   
-H “Content-Type: application/json”   
-d ’{“identity”:“tu@email.com”,“password”:“tuPass”}’

# Luego actualiza el registro (reemplaza RECORD_ID)
curl -X PATCH https://pb.mitechoy.com/api/collections/service_orders/records/RECORD_ID   
-H “Authorization: Bearer TOKEN_AQUI”   
-H “Content-Type: application/json”   
-d ’{“status”:“completed”}’
```

// 06 · Tickets & Email

Sistema de Tickets & Email

Cuando un cliente llena el formulario de "Solicitar Servicio", el sistema hace lo siguiente:

1

Genera un ID de ticket

Formato automático: MTH-2026-XXXX — número aleatorio de 4 dígitos.

2

Intenta guardar en PocketBase

POST a https://pb.mitechoy.com/api/collections/tickets/records. Si responde OK, termina aquí.

3

Fallback: abre cliente de correo

Si PocketBase falla, abre mailto:mitechoy@gmail.com con el formulario pre-llenado. El cliente presiona "enviar" en su app de correo.

4

Muestra confirmación

El formulario se oculta y aparece el número de ticket con instrucciones.

Colección tickets en PocketBase
CampoTipoNotas
ticket_idtextMTH-YYYY-XXXX
nombretextNombre del cliente
telefonotextOpcional
emailemailDel cliente, no tuyo
tipo_serviciotextTutoria / Reparación / etc.
descripciontextDescripción del problema
modalidadtextPresencial / En línea / A domicilio
fecha_deseadatextFecha solicitada por el cliente
notastextNotas adicionales
statustextnuevo / en_proceso / completado
📬 Recibir notificación por email: PocketBase soporta hooks. Ve a Settings → Mail settings y configura SMTP (Gmail App Password). Luego en la colección tickets → Hooks → crea un webhook que envíe email cuando se crea un registro nuevo.
```

```
Gestión de Posts / Blog

Actualmente los posts del blog están hardcodeados en el HTML de index.astro. Puedes moverlos a PocketBase o mantenerlos en código.

Opción A — Editar directo en index.astro (actual)
Cambiar el post destacado html 
<!-- Busca esta sección (~línea 479) -->
```

 class=“post-featured fade-up”>
 src=“URL_DE_IMAGEN” alt=“Descripción”/>
 class=“post-featured-body”>
 class=“post-cat”>Categoría
 class=“post-featured-title”>Título del post
 class=“post-featured-desc”>Descripción corta…

 class=“post-meta”>
📅 01 Abr 2026⏱ 5 min
```
Opción B — PocketBase como CMS de Posts
1

Crea la colección posts en PocketBase

Campos: title category description image_url link date read_time featured (bool)

2

Agrega este bloque JS en index.astro

Reemplaza el HTML estático del blog con una función que haga fetch y renderice las tarjetas.

Fetch de posts desde PocketBase javascript 
// Agrega esto en el bloque <script> de index.astro
```

async function loadPosts() {
try {
const res = await fetch(
PB_URL + ”/api/collections/posts/records?sort=-date&perPage=6”
);
const json = await res.json();
// Renderiza con json.items
} catch {
// Muestra posts estáticos como fallback
}
}
loadPosts();
```
🌟 Recomendación: Para el blog con buen SEO considera migrar a Astro con Content Collections (.md files). Los posts en src/content/blog/ son indexados por buscadores. PocketBase es mejor para contenido dinámico que no necesita SEO.

// 10 · Performance

Performance & Auditoría

Resumen de las mejoras aplicadas en v3.0 y recomendaciones pendientes:

✅
Hero: object-position: top
La foto de portada ya no se corta por arriba. Altura aumentada a 600px desktop / 520px mobile.
✅
Email privacidad
Se eliminó grieverd.him@gmail.com del código público. Todo apunta a mitechoy@gmail.com.
✅
Status Panel integrado
Panel de estado con fetch a PocketBase y fallback estático. Animación LED en tiempo real.
⚡
Fuentes: considera self-hosting
Las Google Fonts (Syne, DM Mono, Noto Serif JP) se cargan de Google. Para máxima velocidad, descárgalas y sírvelas desde tu servidor.
🖼️
Imágenes: convertir a WebP
Las imágenes de Unsplash ya tienen parámetro de calidad. Para hector.jpg usa Squoosh o cwebp para convertir a WebP antes de subir.
📦
Considera dividir en componentes
El archivo index.astro tiene ~1350 líneas. Considera extraer sections a componentes .astro separados para facilitar el mantenimiento.
```

lighthouse https://www.mitechoy.com   
–output=html   
–output-path=./lighthouse-report.html   
–chrome-flags=”–headless”
